## 🧰 접근 방법

- A^B % C는 A^B가 너무 커지므로 그대로 계산할 수 없다
- 거듭제곱의 성질을 이용해 문제를 반으로 쪼개는 분할 정복 방식으로 해결한다

- B가 짝수일 때
  A^B = A^(B/2) × A^(B/2)
  따라서
  A^B % C = ((A^(B/2) % C) × (A^(B/2) % C)) % C

- B가 홀수일 때
  A^B = A^((B−1)/2) × A^((B−1)/2) × A
  따라서
  A^B % C
  = ((A^((B−1)/2) % C) × (A^((B−1)/2) % C) × (A % C)) % C

- B가 1이면 A^1 % C = A % C 이므로 재귀를 종료한다

## BigInt 사용 이유

    - JavaScript Number는 정수를 정확히 표현할 수 있는 범위가 2^53 − 1까지다.
    - 이 문제에서 중간 계산(half × half)은 그 범위를 넘을 수 있다.
    - 범위를 넘으면 값이 틀어질 수 있다.
    - BigInt는 자릿수 제한 없는 정수 타입이라 정확한 계산이 가능하다.
    - 그래서 1629에서는 BigInt를 사용했다.
